Цель АВ-тестирования:

Сравнение двух моделей прогнозирования дефолта по кредитным картам и определение наиболее подходящей для продакшна. В проекте рассмотрены две модели v1 - базовая версия  ,v2 -новая версия. 
Гипотеза: 

Н0: модель v2 не показывает статистически значимых различий в сравнении с v1 
H1: модель v2 показывает значимое улучшение качества прогнозирования в сравнении с v1. 

Cпособ разделения трафика:

Запросы можно разделить случайным образом между двумя моделями, что позволит сравнить модели на похожих группах клиентов. 
v1 - 50 % запросов , v2 - 50% .

Продолжительность теста.

В продакшне в реальных условиях длительность теста может зависеть от разных факторов (например количество кредитных заявок, доля  с дефолтом, бизнес риски) , таким образом, в реальном времени оценка может потребовать длительного периода наблюдения. 


Бизнес-метрики для оценки.

F1 - выбран по причине того,что задача по прогнозированию дефолта является задаченй бинарной классификации. 
Precision - метрика выбрана так как большое количество отказов может упустить хороших клиентов и привести к потерям для банка. Показывает какие клиенты дейтсвительно дефолтного класса. 

Recall - показывает долю дефолтных клиентов чтобы избежать пропуска . 

Метод статистического анализа:

В production-сценарии для анализа результатов A/B-теста можно использовать статистические методы:

z-test — для сравнения долей,
t-test — для сравнения средних значений бизнес-метрик. 

Критерии успешности теста:

1) f1 модели v2 выше чем у модели v1
2) Precision , Recall не ухудшается критически
3) Улучшение статистически значимо
